Reporting a Vulnerability
If you discover a security vulnerability in SysScope, please report it responsibly.
Do not open a public GitHub issue for security vulnerabilities.
Instead, report it privately using one of the following methods:
Open a GitHub Security Advisory through the repository's Security tab.
Email the d28921826@gmail.com.

When reporting a vulnerability, please include:
A clear description of the vulnerability
Steps to reproduce the issue
Potential impact of the vulnerability
Proof-of-concept code or screenshots (if applicable)
Suggested fixes (optional but appreciated)
This helps us reproduce and resolve the issue quickly.
Responsible Disclosure
We ask security researchers to follow responsible disclosure guidelines:
Give the maintainers reasonable time to investigate and fix the issue.
Do not publicly disclose the vulnerability until a fix has been released.
Avoid exploiting the vulnerability beyond what is necessary to demonstrate it.
We will acknowledge valid reports and work to resolve them as quickly as possible.

Security Best Practices
Users of SysScope should follow general security best practices:
Only run the tool in trusted environments.
Avoid executing the tool with unnecessary administrative privileges.
Keep dependencies updated.
Review scripts and configuration files before running them.

Scope
This security policy applies to:
The SysScope source code
Official releases
Configuration and scripts within the repository

Third-party dependencies are maintained by their respective authors.
